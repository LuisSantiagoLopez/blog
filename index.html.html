Flujo de trabajo con Git y GitHub

Pasos del flujo de trabajo

  1. Inicializar repositorio: git init
  2. Añadir archivos al área de staging: git add .
  3. Guardar los cambios: git commit -m "mensaje"
  4. Conectar con repositorio remoto: git remote add origin URL
  5. Subir cambios: git push -u origin main

Comandos más utilizados

ComandoDescripción
git statusVer el estado de los archivos
git logVer el historial de commits
git branchListar o crear ramas
git cloneClonar repositorio remoto
git pullActualizar con cambios remotos

Comandos de Terminal

Comandos comunes usados en clase

  • ls – Lista los archivos del directorio
  • cd – Cambia de directorio
  • mkdir – Crea una nueva carpeta
  • touch – Crea un archivo vacío
  • rm – Elimina archivos
  • clear – Limpia la terminal

Ejemplo práctico

Crear una carpeta, entrar a ella y crear un archivo:

mkdir mi_blog
cd mi_blog
touch index.html
